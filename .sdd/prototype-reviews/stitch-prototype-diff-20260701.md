# Stitch 原型差异清单与落地建议

## 结论

Stitch 原型适合作为本项目的视觉升级参考，尤其是后台骨架、工单详情三栏布局、AI 面板、上传区和 QA 弹窗。但它不能 1:1 落地：部分页面引入了当前 MVP 不支持的能力，部分文案与安全边界冲突，部分调试字段只能保留在 P09 智能问答调试页。

推荐下一步采用“视觉升级优先、功能边界不扩张”的策略：先重做布局、密度、卡片、表格、标签、AI 面板和弹窗样式；所有后端不支持的功能只删不补。

## 可直接采纳

### 全局骨架

- 左侧固定导航 + 顶部账号栏 + 主内容区的结构可以采纳。
- 侧边栏 active 状态采用浅蓝底和左侧蓝色竖条，识别度好。
- 顶部栏保留当前账号、设置、通知、退出，视觉可参考 Stitch。
- 页面背景使用浅灰蓝，主要内容卡片使用白底和细边框。
- 表格行高、状态标签、操作链接的密度更适合客服后台。

### 客服工作台

- 4 个指标卡的布局可采纳：待接工单、处理中、已完成、知识总数。
- 右侧智能助手面板的青绿色 AI 区域可以采纳。
- “待处理工单”表格视觉可采纳，但字段必须使用当前真实接口字段：工单号、标题、发起人、状态、操作。
- 知识库快捷入口卡片可以采纳，保持 QA 库、文档入库、问答调试三项。

### 工单详情

- 三栏布局强烈建议采纳：
  - 左栏：工单基础信息与完成工单。
  - 中栏：会话记录与回复编辑器。
  - 右栏：AI 建议与引用来源。
- 会话气泡、AI 建议卡、引用来源卡片的视觉可以采纳。
- “采用 AI 建议”“发送”“完成工单”的操作分区更清晰，可用于替换当前较平铺的结构。
- AI 面板可以保留“建议回复内容”“引用来源”“需补充信息”，但不得展示内部链路字段。

### QA 管理

- QA 新增/编辑弹窗可以采纳，当前页面已有同类功能，适合增强视觉。
- 表格视觉、启用/停用标签、编辑/删除动作可以采纳。
- 弹窗中的问题、答案、状态字段与当前后端契约一致。

### 文档入库

- 大面积虚线拖拽上传区、选择文件按钮、入库任务列表可以采纳。
- 降级提示条可以采纳，但文案要改成当前项目语义：SQLite vector 不可用时使用 FTS5 / LIKE 降级。
- 表格字段保持当前契约：文档名称、入库状态、切片数量、上传人、更新时间、操作。

### 智能问答调试

- 左侧对话模拟器 + 右侧结果面板的结构可以采纳。
- 引用来源卡片可以采纳。
- 匹配度和上下文提示只允许保留在 P09 调试页，不扩散到工作台和工单详情。

## 需要调整后采纳

### 登录页

- Stitch 登录页整体视觉比当前更克制，可以采纳左右分栏和能力卡片。
- 必须删除“忘记密码”“记住账号”，当前 MVP 不包含找回密码或记住账号。
- 登录名应使用当前系统语义：`agent01` 或“登录名”，不要改成 `agent_001_teamA` 之类不存在的账号。
- 错误提示保留“账号或密码错误”。

### 知识库总览

- Stitch 的指标卡和最近更新列表可以采纳。
- 需要去掉或降级以下内容：
  - 导出报告。
  - 新建知识。
  - 今日检索次数。
  - 昨日新增。
  - 知识分布图表。
- 当前真实接口只支持 QA 数量、文档数量、最近更新时间、最近知识。落地时页面应围绕这些字段设计。

### 设置页

- 账号卡、基础信息卡、服务状态卡可以采纳。
- 模型配置区域必须改成当前安全展示：
  - 数据库。
  - 模型服务。
  - 聊天模型。
  - Embedding 模型。
  - Embedding 维度。
  - API Key 是否配置。
- 不展示 OpenAI / Anthropic，不展示任何 Key 片段，不展示 Base URL。
- 不提供“更新 Key”“模型下拉切换”等当前后端不支持的写操作。

### 文案与数据

- Stitch 示例里很多工单、客户、统计值是静态假数据。落地时必须改回当前接口数据。
- 工单 ID、状态、优先级、客户名、知识数量、模型状态都来自真实 API，不硬编码。
- 品牌名建议统一为“客服机器人”，侧边栏副标题可保留“AI-Powered Support”。

## 暂不纳入

以下能力当前不在 PRD / Plan / API 契约内，本轮 UI 改版不实现：

- 忘记密码、记住账号。
- 通知中心真实交互。
- 导出报告。
- 新建知识统一入口。
- AI 润色 QA。
- 一键填单。
- 多智能体协同。
- 自定义数据报表导出。
- 每月 API 调用额度展示。
- OpenAI / Anthropic Key 管理。
- 模型下拉切换并保存。
- 权限、工号、头像编辑、账号资料编辑。
- 复杂知识分布图表。
- 工单历史时间线展开。
- 意图解析、实体抽取、Vector Search、LLM Generation 耗时瀑布。

这些能力如果未来要做，应作为新需求单独走接口设计和验收。

## 需要后端扩展后才能落地

如果希望完全复刻 Stitch 中更高级的体验，需要新增接口或扩展响应：

- 知识库总览图表：需要知识分类统计接口。
- 今日检索次数、AI 命中率：需要问答调用日志和统计接口。
- 导出报告：需要后端导出任务或文件生成接口。
- AI 润色 QA：需要 QA 润色接口和调用权限。
- 模型选择保存：需要 settings 写接口和配置持久化策略。
- 通知中心：需要通知列表、已读状态和事件来源。
- Intent / Entity / Vector / LLM 耗时瀑布：需要 assistant 调试响应返回内部 trace，并明确只对调试页开放。

当前不建议在本轮视觉改版中新增这些后端能力。

## 推荐改版优先级

### P0：先做全局视觉基线

- 重构整体配色为 Stitch 的浅灰蓝后台风格。
- 统一 8px 圆角、细边框、低阴影。
- 优化左侧导航、顶部栏、按钮、表格、状态标签。
- 保证现有所有页面功能不变。

### P1：重做核心工作流页面

- 客服工作台。
- 工单池 / 我的工单。
- 工单详情三栏布局。

这三类页面是人工验收和真实使用最高频路径，收益最大。

### P2：重做知识库与设置页面

- 知识库总览。
- QA 管理弹窗。
- 文档入库上传区。
- 智能问答调试。
- 账号与基础设置。

### P3：视觉 QA 与响应式修正

- 桌面 1440px 检查。
- 窄屏检查。
- 表格长文本省略。
- 弹窗高度与滚动。
- 所有按钮 disabled / loading / empty / error 状态。

## 验收标准

UI 改版完成后，至少需要满足：

- `VITE_USE_MOCK=false` 下仍命中真实 `/api/...`。
- 不出现 `/src/mocks` 请求。
- 不出现 `[Mock]`、Mock-only、Mock 账号提示。
- 设置页不展示 Key、Token、Base URL、secret、URL 明文或可还原片段。
- 工作台和工单详情不展示 answer_id、confidence、score、prompt、trace、模型版本、上下文数量等内部字段。
- P09 调试页允许展示匹配度和上下文数量，但不展示 prompt、trace、原始检索 JSON 或密钥。
- 人工验收路径仍可通过：登录、工作台、接单、工单详情、AI 建议、回复、完成工单、QA、文档、问答调试、设置退出。

## 推荐实施方式

建议采用分阶段小步改版：

1. 先建立视觉变量和通用组件样式，不改页面逻辑。
2. 再逐页替换模板结构，优先 P02 / P03 / P04 / P05。
3. 每完成一组页面就跑前端 type-check / lint / build。
4. 最后跑完整人工验收，确认真实 API、Mock 退出和安全字段仍满足要求。
